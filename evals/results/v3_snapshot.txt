  CHANGES FROM V2
================================================================================

  System prompt rules refined (agent.py) — no new tools added:

  Rule 1 (Classification questions):
    v2: "enumerate customers first, then read customer_artifacts per customer"
    v3: Explicit ban on customer_artifacts-per-customer for classification.
        Prescribes search-by-pattern instead:
          (a) list_customers() → get account list
          (b) search_artifacts("type A keywords") → find A-group customers
          (c) search_artifacts("type B keywords") → find B-group customers
          (d) Cross-reference; only artifact_full_text() on 1-2 samples per group.
        Rationale: v2 triggered 13 customer_artifacts calls for q6, exhausting
        the tool budget before any synthesis could happen.

  Rule 2 (Competitive/risk questions):
    v2: "use search_artifacts with competitor name or risk vocabulary"
    v3: Prescribes looking up competitor names FIRST via sql_query on the
        competitors table, then searching artifacts for the specific name.
        Rationale: v2 agent searched with wrong customer (HelioFab instead of
        BlueHarbor) because it guessed at vocabulary. Getting the actual
        competitor name ("NoiseGuard") via sql_query makes the artifact search
        deterministic.
    Known bug (see below): competitor "EdgeCollector Co." contains a period
        that survives _sanitize_fts, crashing FTS5 for q5 in v3.

  Rule 3 (Pattern questions):
    v3: Adds Phase 2 geographic check via list_customers(region/country=...) as
        a fallback for customers missed by FTS sweeps. Also adds a 4th example
        FTS query ("approval denied stuck wrong approver").
        Rationale: v2 missed 3/7 Canada customers for q7.

  Rule 9 (Quick-reference sequences):
    Updated to match Rules 1–3 changes above.

================================================================================
  KNOWN BUG IN V3 — q5 crash
================================================================================

  Error   : OperationalError: fts5: syntax error near "."
  Trigger : competitors table contains "EdgeCollector Co." (has a period).
            Agent follows Rule 2: calls sql_query to fetch competitors, gets
            "EdgeCollector Co.", then calls search_artifacts("EdgeCollector Co.").
            _sanitize_fts() strips [-":*()\[\]{}|&^!] but NOT periods.
            The period reaches the FTS5 MATCH clause → SQLite OperationalError.
  Propagation: OperationalError is not caught by LangGraph's ToolNode and
            bubbles out of graph.astream() entirely, so runner.py records
            tools=0 and the exception lands in evaluate_case's error field.
  Fix     : Add '.' to the character class in _sanitize_fts() in tools.py.
